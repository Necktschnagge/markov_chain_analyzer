Detailed Description

Contains all instruction key to use MC Analyzer form command line. Additionally syntax information are given how to use the instructions.

◆ ADD_REW

const auto cli_commands::ADD_REW { "add_rew" }
inlinestatic

Reads edge decorations (rewards) from prism's trew file format. Needs an initialized non-empty markov chain thatt already has a transition for each reward defined in file.

Syntax: add_rew>{id}>{file}>{decoration_index}

Parameters
idid where the markov chain is stored. It must have been initialized before (with reset_mc) and be empty.
filefile path of the *.trew file to read
decoration_indexindex where to store the reward values into each edge.

◆ CALC_EXPECT

const auto cli_commands::CALC_EXPECT { "calc_expect" }
inlinestatic

Calculates for each state of the markov chain the expect of accumulated transition decoration (rewards) until reaching the first state in target set.

Syntax: calc_expect>{mc_id}>{transition_decoration_index}>{target_set_id}>{state_decoration_index}

Parameters
mc_idid where the markov chain is stored.
transition_decoration_indexIndex of transition decorations (rewards) for that the expect should be calculated
target_set_idId to find the set of goal states.
state_decoration_indexIndex of state decorations where the expect should be stored.

◆ CALC_VARIANCE

const auto cli_commands::CALC_VARIANCE { "calc_variance" }
inlinestatic

Calculates for each state of the markov chain the variance of accumulated transition decoration (rewards) until reaching the first state in target set.

Syntax: calc_variance>{mc_id}>{transition_decoration_index}>{target_set_id}>{state_decoration_index}>{state_decoration_expects_index}>{free_transition_decoration}

Parameters
mc_idid where the markov chain is stored.
transition_decoration_indexIndex of transition decorations (rewards) for that the variance should be calculated
target_set_idId to find the set of goal states.
state_decoration_indexIndex of state decorations where the variances should be stored.
state_decoration_expects_indexIndex of state decorations where the expects should be stored. (Note: the algorithm needs to calculate also expects in order to be able to claculate variances.)
free_transition_decorationIndex for storing interim results as transition decorations. Choose a free index that contains values which can be overwritten.

◆ GENERATE_HERMAN

const auto cli_commands::GENERATE_HERMAN { "generate_herman" }
inlinestatic

Generates transitions for Herman's self-stabilizing algorithm and sets all edge decorations at index 0 to 1.

Syntax: generate_herman>{id}>{herman_size}>{target_set_id} It also prints the time that has been spent.

Parameters
idid where the markov chain is stored. It must have been initialized before (with reset_mc) and be empty.
herman_sizeNumber of the processes in ring architecture for herman instance. Must be odd.
target_set_idId of the target set that will be created by the algorithm to store the goal states corresponding to the created markov chain.
